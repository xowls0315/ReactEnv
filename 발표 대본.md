# PHP 블록 에디터 -> React 전환 발표 대본 (main1 기준)

## 0. 오프닝 (30초)
안녕하세요.  
이번 발표에서는 기존 **PHP 디자인 모드의 HTML 블록 삽입 방식**을, React 프로젝트에서 어떻게 **파일 구조 중심으로 전환**했는지 설명드리겠습니다.  
예시는 `main1.html` 블록을 사용했고, 핵심은 **디자인은 유지하면서 구조를 React 친화적으로 재배치**하는 것입니다.

---

## 1. 기존 방식의 특징과 한계 (1분)
기존 `main1.html`은 한 파일 안에 다음 4가지가 섞여 있었습니다.

1. 구조: `<section> ... </section>`
2. 스타일: `<style> ... </style>`
3. 동작: `<script> new Swiper(...) </script>`
4. 콘텐츠 데이터: 이미지 URL, 문구, 메뉴 파라미터

이 방식은 초기 작업은 빠르지만, 블록이 늘어날수록 수정 위치가 불명확해지고 재사용이 어려워집니다.

---

## 2. React 전환 원칙 (핵심) (1분)
React에서는 한 파일에 몰아넣지 않고 **관심사 분리**를 적용했습니다.

- 구조/렌더링 -> `Main1Section.tsx`
- 스타일 -> `Main1Section.css`
- 콘텐츠 데이터 -> `main1Data.ts`
- 레거시 함수 연동 -> `legacyNavigation.ts`

즉, “한 덩어리 HTML”에서 “역할별 파일 구조”로 바꾼 것입니다.

---

## 3. 실제 파일 매핑 설명 (1분 30초)
현재 `src/mainBlock` 기준으로 아래처럼 전환했습니다.

```txt
main1.html              # 원본 참고용
Main1Section.tsx        # JSX 구조 + Swiper 설정 + 이벤트
Main1Section.css        # 기존 style 분리
main1Data.ts            # 배경 이미지/리스트 데이터 분리
legacyNavigation.ts     # GoSubBoard 브릿지
```

`App.tsx`에서는 `Main1Section`만 렌더해서, 블록 단위 컴포넌트가 실제 화면에 올라가도록 구성했습니다.

---

## 4. 변환 과정 상세 (Step-by-Step) (2분)

### Step 1) HTML -> JSX 변환
- `class`를 `className`으로 변경
- 정적 마크업을 컴포넌트 return 내부로 이동

### Step 2) 반복 UI 데이터화
- 배경 슬라이드 이미지, 체크리스트를 `main1Data.ts`로 분리
- JSX에서는 `map()`으로 렌더
- 장점: 문구/이미지 변경 시 컴포넌트 구조는 손대지 않아도 됨

### Step 3) Script 초기화 코드 React 방식으로 이관
- 기존: 페이지 로딩 후 `<script>`에서 `new Swiper(...)` 직접 실행
- 변경: `swiper/react` 컴포넌트와 props로 동일 옵션 적용
- `useRef`로 Swiper 인스턴스를 잡고, `useEffect`에서 모션 감소 설정 처리

### Step 4) `javascript:` 링크 제거 + 브릿지 적용
- 기존 링크: `href="javascript:GoSubBoard(...)"`
- 변경: 버튼 `onClick`에서 `openLegacyBoard(...)` 호출
- 동작:
  - `window.GoSubBoard`가 있으면 기존 PHP 함수 호출
  - 없으면 React fallback 경로(`/contact`, `/portfolio`)로 이동

이 방식이 **점진 전환**에 가장 안전합니다.

---

## 5. 검증 방법 (30초)
전환 후 아래를 확인했습니다.

- `npm run build` 성공
- lint 오류 없음
- 기존 UI/인터랙션(배경 슬라이드, 버튼 동작) 유지

---

## 6. 기대 효과 (40초)
- 유지보수성 향상: 구조/스타일/데이터 수정 위치가 명확
- 확장성 향상: `main2`, `main3`, `main4`도 같은 패턴으로 전환 가능
- 안정성 확보: 레거시 함수와 React 라우트 공존 가능
- 재사용성 증가: 데이터만 바꿔 유사 블록 빠르게 생성 가능

---

## 7. 마무리 멘트 (20초)
정리하면, 이번 전환은 디자인을 바꾸는 작업이 아니라 **코드 운영 방식을 현대화**하는 작업입니다.  
기존 PHP 블록 삽입 구조를 React 파일 구조로 정리해, 이후 유지보수와 확장 속도를 높일 수 있는 기반을 만들었습니다.
